叉乘的右手螺旋定则

发表于 2022-03-17 分类于 学习

写这篇是因为学向量叉乘的时候看到教材说进行叉乘的两个向量和叉乘后的向量符合右手螺旋定则，但，，没说怎么螺旋啊！！非常懵逼！

用右手螺旋定则判断叉乘结果方向

若空间中有向量a和向量b它们叉乘得到向量c，即a x b = c。

（对于a x b，称左边的a为“前向量”，右边的b为“后向量”。这样区分是因为叉乘前后顺序对结果有影响！）

这样判断c的方向：

  1. 右手四指并拢，四指指向“前向量”的方向，拇指张开和四指垂直；

  2. 四指以小于180°的夹角（或者说能够使手趋于握拳的方向）向“后向量”的方向旋转；

    （如果不能满足夹角，或者说不能使手趋于握拳，那就把手上下翻转后再从”1.“做起）

  3. 拇指的方向就是叉乘结果的方向。

举个栗子：

我们有向量a、b进行叉乘，即a x b = c，下面来找c的方向

伸出右手，做”1.“，四指方向就是前向量a的方向。

然后做”2.“尝试以一个小于180°的角度向”后向量“b旋转，但！做不到！

那就把手翻过来，做”1.“

再尝试做”2.“，ok没问题

于是我们找到了c的方向，就是现在的拇指方向。

RGB调色原理与曲线工具

发表于 2022-03-16 更新于 2022-03-17 分类于 学习

原理部分有兴趣的话可以看看，不感兴趣也可以直接看下面的曲线调色部分。

RGB-加减色模型

显色原理

由于人眼视觉的生理原因，将红(R)绿(G)蓝(B)三色按比例混合后可以使人眼产生相应等效色觉（但混合后并不是真的产生了物理意义上的别的颜色）[^1]。

调色原理

简单了解颜色关系

关系 感官上 数学上
相邻色 相似的颜色 色环上相差0~99度的颜色
（对比色） 对比较强的颜色 色环上相差100~179度的颜色
互补色 对比最强的颜色 色环上相差180度（直线正对）的颜色

如何调整颜色

如果想要增加（意思是加深）/减少一种颜色，我们有两种选择：

  1. 增加/减少颜色本身；
  2. 对称地增加/减少颜色的相邻色。

两种方法存在一个区别，就是颜色会变更亮还是更暗，关于怎么判断见下文。

曲线调色

看懂直方图

  • 将一张图片所有的像素取出，按照亮度从低到高的顺序（从左到右）排列，就得到了图片的直方图。

    （灰色表示三色重叠部分）

    ![屏幕截图 2022-03-16 183840](2022.03.16 RGB调色原理与曲线工具/屏幕截图 2022-03-16 183840.png)

  • 通过直方图可以看到整张图像素的亮度分布，找到我们想要调整的像素的亮度，理解曲线是怎么工作的

用曲线调色

选定用哪一条曲线（通道）

为方便理解，我就把红、绿、蓝通道都叫做红、绿、蓝曲线了

  • 调整影调——点曲线（本质是同步调整三色曲线）
  • 调整色调——RGB三色曲线

曲线基本操作

  • 看懂曲线：

    横向对应输入，也就是找到我们想要调整的像素的亮度；纵向对应输出，也就是我们调整后的目标亮度。

  • 具体操作：

    1. 选定输入，打下一个锚点，然后就别动了（指左右横向移动）！

    2. 调整输出值，可以在下面输入数值，也可以鼠标上下纵向拖动。

      （数值越大亮度越高）

※用曲线调整局部颜色及其亮度(划重点！！)

只需要搞懂这个表格！！

  • 首先有我们的红、绿、蓝RGB三原色，改变三原色各自的亮度后混合让我们的眼睛产生了等效色觉。而在只降低其中一种颜色亮度时，画面会呈现出相应颜色的互补色，由此我们有了青、品、黄。

  • 想要增加一种颜色，我们可以增加它本身，也可以对称地增加它的相邻色（改变偶数个数地颜色，对应在色环上向不同方向旋转），减少颜色同理。

  • 每个颜色和自己所在的另一行的颜色有颜色关系，其中：

    和同一竖列的颜色为互补色，和互补色以外的两个颜色为相邻色。

    （利用这个原理调色，要么调整目标颜色，要么调整目标颜色所在的另一行的两个颜色）

说了这么多作铺垫，下面终于到了重点！！

调整局部颜色具体操作：

  1. 找到目标颜色像素的亮度位置（输入）；
  2. 决定要提亮还是减暗，找到对应的那一行（提亮->上面那行；减暗->下面那行）；
  3. 决定要调整的颜色，按调色原理调色（在1.的那一行里，如果有目标色，那就调整目标色；如果没有，那就调整目标色的两个互补色）。

下面举个栗子：

我想让图片高光部分的颜色增亮并偏黄：

  1. 找到高光的位置（曲线中上位置），打下锚点；

  2. 增亮，我在”亮“行里找；

  3. ”亮“行里么有黄色，那就找相邻色，可我也不记得黄色相邻色是啥呀咋办？？emmm，蓝色跟黄色同一列是互补色，排除掉。ok这下我知道了，剩下的红、绿是黄的相邻色，那我就分别在红、绿曲线里打下相应的锚点（可以通过”输入“数值来确保打到同一点），然后把曲线往红、绿的方向纵向拉，高光就变黄了。

记录有什么意义啊

发表于 2022-03-11 更新于 2022-03-15 分类于 碎碎念

记录有什么意义啊

我总在记录，文字啊，照片啊，视频啊等等，各种形式，各有特点，但总归都留存下当时的心情、一些情绪或是想法。我觉得是有一些东西在驱使我记录的，但一直说不出一个明确的原因，或者说，这么做对我的意义。最近在翻看一些以前随笔写下的想法，好像有了一些答案。

人总是在遗忘，或快或慢，不论愿不愿意（写到这里想到了：《追风筝的人》里，哈桑的儿子索拉博发现自己正渐渐忘记死去的父母的样子，感到很不安），但总会有些东西、有些事情是不该忘记，不能忘记，或是不想忘记的。记录和回顾就大概就是在对抗遗忘，在对抗这个熵增的过程，是一种倔强吧。回顾从前，回味那时的特殊心情，当时的天气、温度和空气的味道，那一刻的挣扎或是冲动。

有时我看到以前写的或是拍的东西会惊讶于它内容的有趣或是深刻（这么说好臭屁哈哈哈但，有时真的会想：我怎么写/拍得出来的？？），这感觉像是在观察另一个的独立个体，是很特别的体验，也会提醒我去调整状态，去思考，去变得有意思一点。

记录是很私人的东西，所以也是和自己相处的一种方式。对我来说，和大多数人的交往都是一件耗费精力的事情，所以我很需要这种陪自己的过程，只有这时我才感受到彻底的放松和久违的平静。

目前来说，记录对我的意义大概就是这样。

数组

  1. java数组有内部成员 length 告诉你元素数量。

    （相当于C语言中的 sizeof(array)/sizeof(type of array) ）

  2. 元素个数可以是变量。

字符串

  1. 字符串操作：
    • 比较大小
    • 获得长度
    • 访问某个位置的字符
    • 取得子串（子集）
    • 寻找特定字符
    • ······(等等)
  2. 所有的字符串都是不可变的，对它们的操作结果都是制造新的字符串出来。

对象

    ("a string")：用“a strong”初始化这个对象。

    String s = new String();：创建管理这个对象的变量s。
